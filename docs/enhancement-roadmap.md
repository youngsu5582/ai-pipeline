# AI Pipeline Dashboard - Enhancement Roadmap

> 전체 로드맵 문서. 각 Phase별 상세 명세서는 개별 파일로 존재.
> 마지막 업데이트: 2026-02-06

## 현재 상태 요약

### 구축 완료된 핵심 기능
- Express 서버 (40+ API 엔드포인트)
- 크론 작업 스케줄링 & 실행 엔진
- 파이프라인 체이닝 (edges 기반)
- 실시간 SSE 업데이트
- Electron 트레이 앱 (팝업 입력, 글로벌 단축키)
- Claude Code 세션 관리 & 내보내기
- Obsidian Daily Note 연동
- 노트 & 백로그 (날짜별 조회)
- 하루 시작 / 오늘 보고서 위저드
- GitHub 멀티 계정 활동 수집
- 홈 대시보드 (요약 카드 + 바로가기)
- 작업 탭 서브탭 (목록/이력/통계)

### 핵심 Pain Points
1. **데이터가 흩어져 있음** - 세션, 메모, 작업 이력, GitHub 활동이 각각 별도 탭
2. **분석/인사이트 부족** - 데이터는 쌓이는데 트렌드/패턴을 보여주지 않음
3. **수동 작업 잔존** - 보고서, 회고, 정리를 매번 직접 트리거해야 함

## Phase 1: 통합 타임라인 & 데이터 허브
> 📄 상세 명세: `docs/spec-phase1-unified-timeline.md`

**목표**: 흩어진 데이터를 하나의 타임라인으로 통합하여 "오늘 내가 뭘 했는지" 한눈에 파악

### 1.1 통합 타임라인 뷰 (Unified Timeline)
- 홈 대시보드에 시간축 타임라인 추가
- 데이터 소스: 작업 실행, Claude 세션, 메모, GitHub 커밋/PR, Obsidian 메모
- 시간대별 그룹핑 (오전/오후/저녁)
- 소스별 아이콘 & 색상 구분
- 타임라인 항목 클릭 → 해당 상세 보기로 이동

### 1.2 통합 검색 (Global Search)
- Cmd+K 또는 / 로 열리는 검색 모달
- 모든 데이터 타입을 한번에 검색 (세션, 메모, 작업, 이력)
- 최근 검색 기록
- 검색 결과에서 바로 해당 항목으로 이동

### 1.3 날짜 네비게이션 통합
- 홈 대시보드에 날짜 선택기 (노트 탭처럼)
- 선택한 날짜의 모든 활동을 타임라인으로 표시
- 이전/다음 날 화살표

### 구현 범위
- **수정 파일**: `public/index.html` (홈 탭 확장), `server.js` (통합 타임라인 API)
- **새 API**: `GET /api/timeline?date=YYYY-MM-DD` (모든 소스 통합 반환)
- **예상 작업량**: 중간

## Phase 2: AI 기반 인사이트 & 분석
> 📄 상세 명세: `docs/spec-phase2-ai-insights.md`

**목표**: 쌓인 데이터에서 패턴과 의미를 추출하여 actionable insight 제공

### 2.1 주간 다이제스트 (Weekly Digest)
- 매주 금요일 자동 생성 (또는 수동 트리거)
- Claude가 한 주간의 세션, 메모, GitHub 활동, 작업 이력을 분석
- 출력: 주요 성과, 학습 내용, 다음 주 제안
- Obsidian에 자동 저장 (주간 노트)

### 2.2 생산성 분석 대시보드 (Productivity Analytics)
- 통계 탭 확장 또는 새 "인사이트" 탭
- 시간대별 활동 히트맵 (언제 가장 활발한지)
- 카테고리별 시간 분포 (코딩 vs 리뷰 vs 학습)
- 주간/월간 트렌드 차트
- "이번 주 vs 지난 주" 비교

### 2.3 스마트 서제스션 (Smart Suggestions)
- 홈 대시보드에 AI 기반 제안 카드
- "오늘 아직 메모를 안 남기셨어요" (행동 기반)
- "지난주에 비해 세션이 30% 증가했어요" (트렌드 기반)
- "매주 화요일에 주로 PR 리뷰를 하시네요" (패턴 기반)

### 구현 범위
- **수정 파일**: `server.js` (분석 API), `public/index.html` (인사이트 UI)
- **새 API**: `POST /api/insights/weekly-digest`, `GET /api/insights/productivity`
- **의존성**: Claude API (기존 연동 활용)
- **예상 작업량**: 큼

## Phase 3: 자동화 & 워크플로우 고도화
> 📄 상세 명세: `docs/spec-phase3-advanced-automation.md`

**목표**: 수동 트리거를 줄이고, 더 똑똑한 자동화 파이프라인 구축

### 3.1 조건부 파이프라인 (Conditional Pipelines)
- Edge에 조건 추가: `onSuccess`, `onFailure`, `onOutput(regex)`
- 작업 A 출력값에 따라 분기 (B 또는 C 실행)
- 그래프 뷰에서 조건 시각화

### 3.2 스마트 스케줄링
- "평일에만" 같은 고급 스케줄 옵션 UI
- 작업 의존성 기반 자동 스케줄 조정
- "마지막 실행 이후 N시간" 같은 상대적 스케줄

### 3.3 알림 채널 확장
- Discord Webhook 지원
- macOS 네이티브 알림 (Electron)
- 알림 규칙: 작업별로 다른 채널/조건 설정
- 알림 히스토리 (어떤 알림이 언제 보내졌는지)

### 3.4 외부 트리거 (Webhooks)
- `POST /api/webhook/:token` - 외부에서 작업 트리거
- GitHub Webhook → 특정 작업 자동 실행
- 토큰 기반 인증

### 구현 범위
- **수정 파일**: `server.js` (실행 엔진 확장), `jobs.json` (스키마 확장)
- **예상 작업량**: 큼

## Phase 4: AI/LLM 심화 연동
> 📄 상세 명세: `docs/spec-phase4-ai-deep-integration.md`

**목표**: Claude를 "비서"에서 "분석 파트너"로 격상

### 4.1 자동 메모 분류 (Auto-categorize)
- 메모 저장 시 Claude가 자동으로 태그 부여
- 카테고리: 업무, 학습, 아이디어, 할일, 일상 등
- 태그별 필터링 & 통계

### 4.2 세션 인사이트 (Session Intelligence)
- Claude Code 세션 자동 분석
- "이 세션에서 다룬 주제", "사용한 기술 스택", "해결한 문제"
- 세션 간 연관 관계 자동 탐지
- 프로젝트별 지식 축적 추적

### 4.3 학습 지식 그래프 (Knowledge Graph)
- Obsidian 노트에서 키워드/토픽 추출
- 토픽 간 연결 관계 시각화 (vis-network 활용)
- "이것도 공부해보세요" 추천
- 복습 필요 알림 (간격 반복 학습)

### 4.4 코드 리뷰 어시스턴트
- PR 리뷰 데이터 + Claude 분석
- 리뷰 패턴 분석 (자주 지적하는 항목, 자주 지적받는 항목)
- 리뷰 체크리스트 자동 생성

### 구현 범위
- **새 API**: `POST /api/ai/categorize`, `GET /api/ai/knowledge-graph`
- **의존성**: Claude API, Obsidian vault 접근
- **예상 작업량**: 매우 큼

## Phase 5: 플랫폼 확장
> 📄 상세 명세: `docs/spec-phase5-platform-extension.md`

**목표**: 언제 어디서든 접근 가능한 플랫폼으로 확장

### 5.1 반응형 모바일 UI
- 현재 UI를 모바일에서도 사용 가능하게
- 핵심 기능: 타임라인 조회, 빠른 메모, 작업 실행
- PWA (Progressive Web App) 지원

### 5.2 위젯 시스템
- 홈 대시보드 위젯 커스터마이징
- 드래그 & 드롭으로 위젯 배치
- 위젯 종류: 타임라인, 통계 차트, 퀵 메모, 세션 요약, 캘린더

### 5.3 플러그인 아키텍처
- 서드파티 연동을 위한 플러그인 시스템
- 플러그인 타입: 데이터 소스, 알림 채널, 분석기
- 설정 UI 자동 생성

### 5.4 서버 아키텍처 개선
- server.js 모듈화 (routes/, services/, utils/)
- SQLite 전환 (JSON 파일 → DB)
- 비동기 I/O 전면 적용
- Docker 배포 지원

### 구현 범위
- **예상 작업량**: 매우 큼 (장기 과제)

## 권장 구현 순서

```
Week 1-2: Phase 1 (통합 타임라인 + 검색)
  └── 데이터 통합의 기반, 모든 후속 기능의 전제

Week 3-4: Phase 2.1 + 2.2 (주간 다이제스트 + 생산성 분석)
  └── 축적된 데이터에 의미 부여

Week 5: Phase 4.1 (메모 자동 분류)
  └── 작지만 체감 효과 큰 AI 연동

Week 6-7: Phase 2.3 + 4.2 (스마트 서제스션 + 세션 인사이트)
  └── AI 기반 능동적 제안

Week 8+: Phase 3 (자동화 고도화)
  └── 워크플로우 개선

Long-term: Phase 4.3, 5 (지식 그래프, 플랫폼 확장)
```
